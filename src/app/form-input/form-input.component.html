Form Input Component

mc-form-input

Component Input Directives:

Directive Type Description
label string The string that will be displayed along side the input, to note what goes inside the input.
startingCol string A number string of the starting column. If the grid has 3 columns and you want it to start at 2, you put 2 here.
columnSpan number The number of columns that the input will take up. Defaults to 1.
rowSpan number The number of rows that the input will take up. Defaults to 1.
labelWidth string Sets the width of the label (in the current column). Setting it to be smaller than the space the text currently takes up will wrap the label text. Can be set in units of px, em, rem, vw, and vh. (eg. labelWidth="100px", labelWidth="2rem", labelWidth="20vh")
alignVertical boolean


Question Icon Tooltip

Input Directives for adding an icon to the end of an input that contains extra information in a hover tooltip.

Directive Type Description
questionText string | TemplateRef<any> A provided string or TemplateRef that populates a ? icon with a popover at the end of the input. If a value is provided the icon will be created automatically.
questionTitle string If a questionText value is also provided, the tooltip/popover box creates a title area with this string as the title.
questionPlacement 'top' | 'left' | 'right' | 'bottom' Determines from which direction the tooltip/popover will extend from the icon. Useful for icons that are pushed up against the edge of the window. Default: 'top'
containerBody boolean If true, the popover container is appended to the html <body> element instead of the icon element. This can solve z-index and accessibility issues. Defaults to true
tooltipLocation 'over-label' | 'icon-end' | 'label-end' By Default, an input-button icon wil appear at the end of the input. ('icon-end')
If this value is 'over-label', hovering over the label will display the popover.
If this label is label-end, an icon will appear at the end of the label. NOTE: in order for this to look good, alignVertical must be set to true.


Error Messages and Highlighting

Input directives used to display error messages when an input is invalid.

Use the control and validationMessages inputs for better control over reactive forms.

Use the errorMsg and invalidInput inputs for template-driven forms.

Directive Type Description
control FormControl Provide a FormControl to access the FormControl's errors object and display the corresponding validation message in validationMessages
validationMessages {{ '{' }} [x: string]: string {{ '}' }} Provide an object where the key is one of the FormControl's potential errors, and the value is the message you want displayed when that error is true.
errorMsg string | string[] A string message to display whenever invalidInput is true. A string array could originally have been passed in for multi-inputs, but that property is now deprecated. Use at your own risk.
invalidInput boolean | boolean[] If true, display the errorMsg passed in. Set this property up to change depending on registered changes from the input.
